JSSP 문법 살펴보기

JSSP 문법은 다양한 플랫폼의 소스 코드로 변환되므로 일반적인 프로그래밍 언어보다는 많은 타입이 존재합니다. 그렇지만 이러한 복잡성에도 불구하고 직관적이고 쉽게 배울 수 있는 구조로 설계되어 있습니다.

Domain 정의​

Domain은 데이터베이스를 위한 타입입니다.

domain Email varchar(32);
domain Nickname varchar(128);
domain UserID bigint;
domain Url varchar(1024);

domain 타입은 데이터베이스를 설계하고 query를 작성하는데 사용할 수 있습니다.

테이블 설계​

table User {
 user_id UserID auto;
 email   Email;
 nickname Nickname;
 profile_url Url;
 create_at Datetime;
 key(user_id);
 unique index(email);
 index(nickname);
}

위 코드 조각은 JSSP 문법을 사용하여 "User" 테이블을 정의하고 있습니다. 여기에는 다음과 같은 컬럼이 포함되어 있습니다:

  • user_id: UserID 타입이며, 자동 증가(auto) 설정이 되어 있습니다.
  • email: Email 타입의 이메일 주소를 저장합니다.
  • nickname: Nickname 타입의 사용자 닉네임을 저장합니다.
  • profile_url: Url 타입의 프로필 URL을 저장합니다.
  • create_at: Datetime 타입의 생성 날짜와 시간을 저장합니다.

그리고 다음과 같은 제약 조건과 인덱스가 있습니다:

  • user_id를 기본 키로 설정합니다.
  • email에 대한 고유 인덱스를 생성하여 동일한 이메일 주소가 중복되지 않도록 합니다.
  • nickname에 대한 일반 인덱스를 생성하여 닉네임을 기반으로 한 검색을 빠르게 수행할 수 있게 합니다.

이 정의는 데이터베이스 테이블을 생성하거나 쿼리를 작성하는 데 사용될 수 있으며, 특정 데이터베이스 시스템에 따라 변환될 수 있습니다.

UserDetail 엔터티 정의​

JSSP에서 엔터티는 데이터베이스의 CRUD 작업을 수행하기 위해 사용되는 데이터 구조를 정의합니다. 아래는 UserDetail 엔터티의 예입니다.

entity UserDetail {
  userId UserID;
  email Email;
  nickname Nickname;
  profileUrl Url;
  createAt Datetime;
}

이 엔터티는 다음과 같은 속성을 포함합니다:

  • userId: UserID 도메인 타입의 사용자 ID를 나타냅니다.
  • email: Email 도메인 타입의 사용자의 이메일 주소를 나타냅니다.
  • nickname: Nickname 도메인 타입의 사용자의 닉네임을 나타냅니다.
  • profileUrl: Url 도메인 타입의 사용자의 프로필 URL을 나타냅니다.
  • createAt: Datetime 도메인 타입의 사용자 계정의 생성 날짜와 시간을 나타냅니다.

이 엔터티는 특정 쿼리에서 사용자 상세 정보를 가져오는 데 사용될 수 있습니다.

selectUser 쿼리 정의​

selectUser 쿼리는 특정 사용자 ID를 기반으로 사용자의 상세 정보를 선택하는 데 사용됩니다. 이 쿼리는 각 플랫폼마다 실행가능한 코드로 변환됩니다. 예를 들어, Spring Boot에서는 XML로 변환되고, 다른 언어에서는 코드로 변환됩니다.

query selectUser(userId UserID) UserDetail {
  select user_id as userId,
    email,
    nickname,
    profile_url as profileUrl;
    create_at as createAt
  from user
  where user_id = :userId;
}

이 쿼리는 다음과 같은 매개 변수를 사용합니다:

  • userId: UserID 타입의 사용자 ID를 나타냅니다.

반환값은 UserDetail 엔터티에 정의된 대로 됩니다.

HTTP 요청과 응답 코드​

Error 정의​

코더스에서 사용되는 에러 코드를 정의하는 방법은 아래와 같습니다. 이러한 에러 코드 정의는 API 명세서 문서를 통해 어떤 에러 코드를 리턴하는지 알 수 있으므로 유지보수 측면에서 매우 강점이 있습니다.

define error {
   Success 0 '성공'
   Failed   1 '실패'
}

Struct 정의​

struct는 요청과 응답에 사용되는 데이터 타입입니다. 각 플랫폼으로 변환될 때 각 플랫폼에 맞는 JSON 직렬화 코드를 자동으로 작성하므로, 직렬화 코드 작성의 번거로움을 감소시켜줍니다. UserInfo 구조체는 사용자의 아이디, 이메일, 닉네임 등의 정보를 담고 있습니다.

struct UserInfo {
  userId UserID;
  email Email;
  nickname Nickname;
}

Opcode 정의​

이 코드는 사용자 정보를 수신하는 HTTP GET 요청을 정의합니다. controller로 지정된 User는 서버 백엔드의 MVC 패턴에서 컨트롤러로 변환됩니다. 또한 baseUrl로 설정된 경로를 기반으로 요청 URL이 구성됩니다.

define opcode[controller=User, baseUrl="/api/v1"] {
  [method=get, action='/user', noauth]
  getUser 100 '사용자 정보 수신'
}

이렇게 정의된 getUser 오퍼레이션은 /api/v1/user 경로로 GET 요청을 받아 사용자 정보를 수신하는 기능을 담당합니다. noauth 옵션은 이 요청이 인증을 필요로 하지 않음을 나타냅니다.

Request와 Response 정의​

이 코드는 사용자 정보를 가져오기 위한 요청(req)과 응답(res)을 정의합니다. getUser라는 이름으로 요청과 응답을 처리하며, 요청에는 이메일(email)을 인자로 받고, 응답에는 사용자 정보(userInfo)를 반환합니다.

req getUser {
  email Email;
}

res getUser {
 userInfo UserInfo;
}

req 키워드는 요청 데이터를 정의하고, res 키워드는 응답 데이터를 정의합니다. 여기서 userInfo는 이전에 정의된 struct 타입을 사용하며, 사용자의 아이디, 이메일, 닉네임 등의 정보를 포함할 수 있습니다.

Handler 작성​

아래의 코드는 사용자 정보를 가져오는 getUser 핸들러를 정의합니다. 이 코드는 각 플랫폼에 맞게 소스 코드로 변환되며, 요청 객체를 검증한 후, 쿼리를 통해 사용자의 상세 정보를 가져오고 응답 객체에 설정합니다.

handler getUser {
  func execute(req @req) (res @res) {
     if(req.invalid()) {
        res.code = @error.Failed;
        return;
     }

     var userDetail = @query.selectUser(req.email);
     res.userInfo = UserInfo();
     res.userInfo.set(userDetail);
     res.code = @error.Success;
     return;
  }
}
  1. 요청 검증: 요청 객체가 유효하지 않으면, 응답 코드로 실패(@error.Failed)를 설정하고 반환합니다.
  2. 사용자 정보 조회: @query.selectUser(req.email)을 통해 사용자의 상세 정보를 가져옵니다.
  3. 응답 설정: 응답 객체의 userInfo 필드를 새 UserInfo 객체로 설정하고, 가져온 사용자 상세 정보를 설정합니다.
  4. 성공 응답: 최종적으로 응답 코드로 성공(@error.Success)를 설정하고 반환합니다.

이러한 방식으로 작성된 코드는 각 플랫폼에 맞게 소스 코드로 변환되어, 플랫폼 간의 호환성을 제공하고 개발의 편리성을 높여줍니다.
